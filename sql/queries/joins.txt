//joins-соединение, union - объединение
//inner join - в выборку попадут только те записи из левой таблицы, ключи которых есть в правой таблице

//left outer join - в выборку попадут все записи из левой таблицы, а те что не имеют ключей в правой таблице будут NULL
//right outer join - в выборку попадут все записи из правой таблицы, а те что не имеют ключей в левой таблице будут NULL

//full outer join - все записи из двух таблиц, не сопоставимые будут NULL

//cross join - сопоставление все записей друг другу из двух таблиц(5 в одной, 5 в другой = выборка из 25 записей)

//self join - есть таблицы, где внешний ключ ссылается на первичный ключ этой же таблицы(у работника может быть менеджер - такой же работник)
